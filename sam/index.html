SOBRE HTTP

http (del inglés HyperText Transfer Protocol o HyperText Transfer Protocol) es el protocolo de transferencia de mensajes de la World Wide Web, es decir, el código establecido para que el ordenador solicitante y el que tiene la información solicitada "hablen" el mismo idioma a la hora de transferir información en la red Con http se establecen métodos para la sintaxis y semántica de los datos (forma y significado) para establecer la comunicación entre los distintos elementos que componen la arquitectura de Internet: servidores, clientes, proxies. Fue desarrollado en 1999 por el World Wide Web Consortium en colaboración con Internet Engineering Task Force. Es un "protocolo" infinito, es decir, no registra visitas anteriores, sino resúmenes. La información sobre visitas anteriores se almacena en el sistema en las llamadas "cookies", almacenadas en el cliente. http ha pasado por varias versiones hasta llegar a la actual a principios del siglo XXI, denominada HTTP/2. El primer intento se hizo en 1991, y se produjeron varios modelos en 1996, 1999, 2000 y finalmente el actual en 2015.

PREGUNTAS FRECUENTES SOBRE EL PROTOCOLO HTTP

¿Qué es http?

Por https entendemos HyperText Transfer Protocol Secure o Protocolo Seguro de Transferencia de Hipertexto, que no es más que una versión segura de http, es decir, el mismo tipo de protocolo que se basa en la creación de una forma cifrada de envío del mensaje. , lo que lo hace más adecuado para ciertos problemas de privacidad (como contraseñas y nombres de usuario).

¿Qué utilidad tiene el protocolo http?

Es un registro "ilimitado", es decir, no registra los tiempos anteriores, sino el resumen. La información sobre visitas anteriores se almacena en un sistema llamado "cookies", almacenado en el cliente. http ha pasado por varias versiones hasta llegar a la versión actual a principios del siglo XXI, llamada HTTP/2. El primer intento se hizo en 1991, y se hicieron diferentes modelos en 1996, 1999, 2000 y finalmente el disponible en 2015.

¿Cómo trabaja el protocolo HTPP?

La funcionalidad de http se basa en un protocolo de solicitud-respuesta entre un servidor web y un "servidor" o cliente que envía una solicitud de datos. El cliente puede ser un navegador en particular cuando intentamos acceder al sitio web, o rastreadores web (web crawlers o arañas) que lo ven. Para ellos, el servidor brinda una solución preparada en tiempo y con una lista de metadatos, que establece los procedimientos de inicio, desarrollo y cierre para la transferencia de información. Estos son "métodos de solicitud", es decir, comandos que activan la ejecución de ciertos objetos, cuyos archivos están en el servidor.
